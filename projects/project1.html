Project Overview

The Higher-Lower Game was my first project at ROC Mondriaan. The goal was to create an interactive browser game using only the fundamentals: HTML for structure, CSS for styling, and JavaScript for all game logic. No frameworks, no libraries — just plain web development.

I chose to build a single-player experience where a random number is drawn each round and the player bets credits before guessing whether the next number will be higher or lower. The game has clear win and lose conditions, making it a complete, self-contained product.

How It Works

Game Loop
  • A random number is revealed to the player.
  • The player enters a bet (up to their current credit balance).
  • The player clicks "Higher" or "Lower".
  • A new number is drawn and compared to the previous one.
  • Credits are won or lost based on the outcome and the bet size.
Win & Lose Conditions
  • Win: Reach 3,600 credits — you beat the game.
  • Restart: Drop to −2,600 credits — the game resets.
  • Starting balance: 1,000 credits.
  • The game is infinitely replayable after a restart.

Key Features

  • Dynamic credit system — bet any amount within your current balance.
  • Real-time feedback — win/lose animation and credit update after each round.
  • Input validation — prevents invalid bets (negative amounts, bets exceeding balance).
  • Win & restart detection — automatically triggers end-game messages.
  • Clean, responsive layout — works on desktop and mobile.
  • No external dependencies — pure HTML, CSS, and JavaScript.

What I Learned

JavaScript Fundamentals

DOM manipulation, event listeners, conditional logic, random number generation, and state management across rounds.

UI/UX Thinking

Designing clear feedback for the player — colour changes, messages, and credit tracking — to make the game intuitive without any instructions.

Debugging

Tracing logic errors in bet validation and edge cases (e.g., equal numbers, zero-balance bets) using browser DevTools.
