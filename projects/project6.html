Project Vision & Business Goals

Core Purpose

Provide a secure, easy-to-use budgeting tool for students that tracks income, fixed costs, variable spending, and presents basic financial insights — all within a clean MVC architecture.

Secondary Purpose

Offer a small community area (forum) and a contact channel to practice full-stack CRUD workflows, while demonstrating professional Symfony patterns to recruiters and teaching staff.

Key Features

  • User registration, login & profile editing with picture upload
  • Budget categories (Income, Fixed Costs, Variable Spending) with limits
  • Transaction CRUD per category
  • Community forum with threads, replies & pagination
  • Contact form for authenticated users (stored in DB)
  • Symfony Security — hashed passwords, CSRF, route-level authorization
  • Manual testing completed — test cases documented in test rapport
  • Dashboard stub present — charts & visualisation not yet built

Strengths vs Gaps

  • Passwords hashed, CSRF active, safe file uploads
  • Doctrine ORM — normalised DB with FK constraints
  • Symfony Form types provide server-side validation
  • KnpPaginatorBundle — forum pagination built in

  • NavigationBarController + reusable Twig base layout
